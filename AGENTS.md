# Repo Instructions

## Purpose
- This repo contains a static Hyperliquid / HIP3 dashboard intended for GitHub Pages.
- Data is refreshed locally, then embedded directly into the generated HTML.

## Refresh Workflow
- Main refresh command:
  `cd /Users/mattmaximo/Code/hype-dash && ALLIUM_API_KEY=... .venv/bin/python build_data.py`
- That command updates:
  - `data/dashboard-data.json`
  - `index.html`
  - `dist/index.html`
  - copied static assets under `dist/`

## Data Sources
- Exact daily fee and volume aggregates:
  - Allium `hyperliquid.metrics.overview`
- Exact latest active market rows:
  - Allium `hyperliquid.dex.trades`
- Revenue and burn totals:
  - Artemis Snowflake `ART_SHARE.HYPERLIQUID.EZ_METRICS`
- Historical HIP3 full-fee counterfactual:
  - Estimated from Artemis category/token share model

## Known Caveats
- Market table values are the latest available active row per market over the recent Allium window, not necessarily the same calendar day for every market.
- Some markets can still show `No recent trade` if there is no recent Allium trade row.
- Revenue and burn splits remain estimated because Allium does not publish those totals directly.

## GitHub Pages
- Deploy artifact is `dist/`.
- GitHub Actions workflow publishes `dist/` to Pages on pushes to `main`.

## Editing Guidance
- Edit layout/content in `index.template.html`, not the generated `index.html`.
- `index.html` is generated by `build_data.py`.
